# /goal 模板

本文档用于本项目日常开发时快速启动 `/goal`。`AGENTS.md` 负责长期规则，`/plan` 负责需求收敛，`/goal` 负责当前这一次持续执行的任务。

## 使用原则

建议固定写法：

```text
动作 + 具体对象 + 所属链路环节 + 不改什么 + 验收 + 测试/汇报
```

如果任务还不能明确到单一链路环节，先用 `/plan`，不要直接开 `/goal`。

本项目的 `/goal` 默认遵守这些边界：

- 本地 AS译林副本和 OCR / 翻译 / 校对工作台不要混在一个目标里改。
- AS译林本地副本默认只验证和修改本地链路，不影响线上 `https://asyilin.cpolar.top/`。
- 发现问题时先定位到具体链路环节，再决定是否改配置、adapter、聚合器或业务代码。
- 一个线程只挂一个主目标；范围过大时拆成多个阶段性 `/goal`。
- 长说明超过几千字符时，把细节放进文档，让 `/goal` 指向文档。

## bug 修复

```text
/goal 修复 [AS译林本地副本 / OCR翻译工作台] 中 [页面/接口/容器/脚本] 的 [具体问题]。先按 [LLM API key 链路第X步 / OCR翻译链路第X步] 定位，只修改 [文件/模块/配置]，不修改 [不相关模块]，不影响线上 https://asyilin.cpolar.top/。补一个回归验证，运行 [命令] 并汇报结果。完成标准：[复现步骤] 下问题消失，且 [上游环节] 保持正常、[下游环节] 可以继续走通。
```

## 重构

```text
/goal 重构 [模块/文件/链路环节]，目标是 [拆职责/降耦合/提可测性]，但不改变 [外部 API / 页面行为 / 容器名 / 缓存结构 / 数据格式]。范围只限 [文件或目录]，不触碰 [AS译林链路 / OCR翻译链路] 的其他模块。补或更新相关测试，运行 [命令] 并汇报结果。完成标准：现有功能按原复现步骤仍然正常，代码职责边界更清晰，且没有引入新的 OCR、翻译或 LLM 调用回归。
```

## 上线前收敛

```text
/goal 为 [本地 AS译林副本 / OCR翻译工作台] 做上线前收敛，只处理 [最多 3 项明确问题]，不新增功能、不改依赖版本、不改线上部署。需要完成 [启动 / 登录 / 配置保存 / 模型调用 / OCR / 翻译 / 导出] 中与本次范围相关的冒烟验证，运行 [命令] 并汇报结果。完成标准：[访问地址或入口] 可正常打开，关键链路全部走通，阻塞上线的问题清零，剩余问题都有明确记录且不影响主流程。
```

## 功能开发

```text
/goal 在 [AS译林本地副本 / OCR翻译工作台] 新增 [具体能力]，服务于 [具体场景]。保持 [现有 API / 部署方式 / OCR主流程 / 翻译主流程] 不变，新增逻辑只落在 [模块/目录]，并在失败时明确区分 [前端配置 / 后端 / 适配器 / 外部模型 API / 文本抽取 / OCR fallback]。补最小可用验证，运行 [命令] 并汇报结果。完成标准：用户按 [操作步骤] 可以完成 [结果]，成功路径稳定，失败路径提示清楚，且不破坏现有链路。
```
