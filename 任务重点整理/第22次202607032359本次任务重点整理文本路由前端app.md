# 本次任务重点

- 梳理翻译输入来源链路。
- 修复右栏翻译固定读取 OCR 文本的问题。
- PDF 当前页优先从文本层提取文字，再作为翻译输入。
- DOCX 支持浏览器端读取 `word/document.xml` 提取正文；旧版 `.doc` 明确提示转为 `.docx` 或 PDF。
- 中栏状态从单纯 OCR 改为“文本 / OCR”，区分 `PDF 文本层`、`Word 文本`、`BDRC OCR` 和人工校对文本。

## 下一步行动

- 用用户当前可编辑 PDF 重新上传测试。
- 确认中栏显示 `PDF 文本层`，文件状态显示 `文本已提取` 或 `文本提取进行中`。
- 点击“翻译”后确认右栏不再使用旧 OCR 错字。

## 需要跟前端交流的内容

- 翻译按钮现在会先解析当前页真实源文本。
- PDF 文本层优先级高于 BDRC OCR。
- 手动校对文本仍保留最高优先级，避免覆盖人工修正。
- DOCX 支持是轻量前端解析，不支持旧 `.doc` 二进制格式。

## 需要跟前端 app 同步的内容

- 文件路由顺序：PDF 文本层 / DOCX 文本 / Markdown/TXT 直读 / 图片或扫描 PDF OCR fallback。
- 中栏已不再等同于 OCR，它是“文本 / OCR 结果”承载区。

## 需要跟管理后台同步的内容

- 无。

## 需要跟后端服务器同步的内容

- 无需改 OCR 或翻译后端。
- 翻译请求新增 `source_kind` 和 `source_label` 字段，后端可忽略，不影响兼容。

## 需要跟数据库同步的内容

- 无。
